Deploying App Updates

All code lives in a single file: index.html in the c3jumpw/Rhis-Kitchen GitHub repo. GitHub Pages serves it publicly.

How to deploy a code change

  1. Edit the local index.html file (or receive updated file from MK)
  2. Go to github.com/c3jumpw/Rhis-Kitchen
  3. Click index.html → click the pencil (Edit) icon → paste new content → Commit changes
  4. Or: use GitHub Desktop to drag-and-drop the file and push
  5. Wait 60 seconds → clear browser cache (Ctrl+Shift+R) → verify at the live URL

What's baked in vs configurable

ItemHow to change it
Webhook URLsEdit CFG.zap1–8 in the index.html source

Employee passcodesEdit Google Sheets → Employees tab

Zap 1 — Step-by-step structure

  1. Webhooks → Catch Hook (trigger)
  2. Google Sheets → Lookup Dropdowns (Payment Method index)
  3. Google Sheets → Lookup Dropdowns (Pay Status index)
  4. ClickUp → Create Task (returns Task ID used as Order ID everywhere)
  5. Twilio → SMS to customer PENDING
  6. SendGrid → Email to customer PENDING
  7. Twilio → SMS alert to Jeffrey PENDING
  8. Google Sheets → Create Orders row (uses ClickUp Task ID as Order ID)
  9. Formatter → Convert to uppercase (Order ID)
  10. Google Sheets → Lookup Customers row (create if not found)
  11. Paths → New Customer (Data Was Found = false) → Delay 0min
  12. Paths → Returning Customer (Data Was Found = true) → Math +1 orders, +total spent → Update Customers row

Troubleshooting Common Issues
  • 🔴 Order doesn't appear in ClickUp
    Likely cause
    Phone number format rejected OR ClickUp 400 error
    Fix
    Check Customer Phone field is Text type (not Phone type)
  • 🔴 Zap 1 errors with 'Value not option index'
    Likely cause
    Payment dropdown not using index lookup
    Fix
    Ensure Steps 2/3 use Dropdowns tab lookup → Column C (ClickUp Index), not Column B (Label)
  • 🔴 Sheets Orders row not created
    Likely cause
    Zap 1 failed at ClickUp step, skipping Sheets step
    Fix
    Fix ClickUp error first — Sheets step is downstream in Zap 1
  • 🟡 Customer profile name disappears
    Likely cause
    Tag added before order loaded into APP.customers
    Fix
    Fixed in current code — localStorage now preserves names
  • 🟡 App loads blank orders on refresh
    Likely cause
    Sheets Orders CSV not published
    Fix
    File → Share → Publish to web → Orders → CSV → Publish
  • 🟡 Webhook CORS error / Failed to fetch
    Likely cause
    Browser blocking cross-origin POST with JSON header
    Fix
    Fixed — app uses text/plain which skips CORS preflight
  • 🟡 ClickUp custom fields empty after sync
    Likely cause
    cuFieldMap not loaded (Fetch Statuses not clicked)
    Fix
    Auto-loads on page open — click Fetch Statuses in Settings if still empty
  • 🟡 Blast count shows wrong number
    Likely cause
    Tags or customer data loaded async after count renders
    Fix
    Tags and orders now call updateBlastCount() on load — fixed in current code
  • 🟡 Duplicate orders in Sheets
    Likely cause
    'Create if not found' set to Yes on Lookup step in Zap 2
    Fix
    Set Create if not found to NO in every Zap 2 Sheets Lookup step
  • 🟡 Tags lost after page refresh
    Likely cause
    No localStorage — masterTags reset every load
    Fix
    Fixed — APP.masterTags and APP.customers now saved to localStorage

Routine Maintenance
📅 Weekly — Jovell
  • Review the Blasts tab in Sheets — ensure sent blasts are logged
  • Check Orders tab for rows without Order IDs — these are sync failures, flag for MK
  • Archive completed ClickUp tasks (move to a Completed archive folder)
📆 Monthly — Jeffrey / MK
  • Review and update employee passcodes in Sheets → Employees tab
  • Update Dropdowns tab if payment methods or pay status options change
  • Check Zapier task usage — free plan allows 100 tasks/month, Starter allows 750
  • Review customer tags — remove outdated ones via Settings → Tag Manager → Push Tags to Sheets
  • Pull Cash App and Zelle transaction history → add new food customers to Customers tab
🍽 Adding a new menu item
  1. Open the app → Menu tab → Quick Add Item
  2. Fill in: Name, Type, Price, Cost, Ingredients (comma-separated), Servings, Description
  3. Click Update Menu to push to Sheets
  4. Menu CSV auto-reloads next time anyone opens the app
👤 Adding a new employee
  1. Open Google Sheets → Employees tab
  2. Add a new row: Employee ID (any unique number), Name, Passcode
  3. The CSV is already published — changes are live within 60 seconds
